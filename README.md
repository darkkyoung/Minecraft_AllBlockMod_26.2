# All Blocks Challenge

**All Blocks Challenge**는 마인크래프트 생존 모드에서 가능한 많은 블록을 도감에 등록하는 챌린지 모드이다.

플레이어는 생존 플레이를 하면서 블록을 획득하고, 최초로 획득한 블록은 도감에 등록된다. 등록된 블록은 플레이어별 색상으로 표시되며, 모드에 따라 혼자 도전하거나, 협동하거나, 다른 플레이어 또는 팀과 경쟁할 수 있다.

단순한 블록 수집을 넘어서, 진행률 이벤트, 일수 이벤트, 사망 패널티, 랭킹, 도감 UI, 드래곤 알 추격전 등 방송용 콘텐츠에 어울리는 다양한 규칙을 포함하는 것을 목표로 한다.

---

## 개발 환경

* Minecraft: `26.2`
* Mod Loader: `Fabric`
* Language: `Java`
* Mod ID: `allblocks`
* Mod Name: `All Blocks Challenge`
* Package: `com.darkk0729.allblocks`

---

## 핵심 목표

이 모드의 핵심 목표는 다음과 같다.

```text
블록 획득
→ 도감 등록
→ 플레이어별 소유권 표시
→ 진행률 및 랭킹 반영
→ 일정 조건마다 이벤트 발생
→ 제한 시간 또는 모드별 종료 조건에 따라 결과 확정
```

---

## 게임 모드

All Blocks Challenge는 최종적으로 다음 4가지 모드를 지원하는 것을 목표로 한다.

| 모드         | 설명                                    |
| ---------- | ------------------------------------- |
| Solo       | 혼자서 100일 동안 가능한 많은 블록을 수집하는 모드        |
| Co-op      | 여러 플레이어가 하나의 도감을 함께 채우는 협동 모드         |
| Team Race  | 팀을 나누어 팀별 도감 등록 수를 경쟁하는 모드            |
| Block Race | 개인전 경쟁 모드. 가장 많은 블록을 도감에 등록한 플레이어가 우승 |

## 기본 시작 흐름

기본 명령어는 다음과 같다.

```mcfunction
/allblocks start
```

초기 구현에서는 아래처럼 모드를 직접 지정해서 시작할 수 있도록 한다.

```mcfunction
/allblocks start solo
/allblocks start coop
/allblocks start team_race
/allblocks start block_race
```

추후에는 `/allblocks start` 입력 시 모드 선택 UI를 띄우거나, 플레이어 앞에 선택용 블록을 보여주는 방식으로 확장할 수 있다.

챌린지가 시작되면 다음 데이터가 초기화된다.

* 도감 등록 상태
* 플레이어별 점수
* 플레이어별 색상
* 플레이어별 표시 이름
* 사망 횟수
* 진행률 이벤트 기록
* Day 이벤트 기록
* 챌린지 타이머
* Block Race의 드래곤 알 상태

## 종료 명령어

진행 중인 챌린지는 다음 명령어로 강제 종료할 수 있다.

```mcfunction
/allblocks stop
```

챌린지가 종료되면 다음 상태가 된다.

* 도감 업데이트 중단
* 블록 수집 판정 중단
* 이벤트 발생 중단
* HUD 정지
* 스톱워치 정지
* 랭킹 고정
* 드래곤 알 슬롯 잠금 해제
* Egg Carrier 상태 제거
* 발자국 흔적 제거

이후 다시 `/allblocks start`를 입력하면 기존 데이터는 리셋되고 새 챌린지가 시작된다.

## 플레이어 설정

챌린지 시작 시 각 플레이어에게 색상이 자동 배정된다.

기본 색상 후보는 다음과 같다.

```text
red, orange, yellow, lime, green, cyan, blue, purple, pink, white
```

색상은 추후 명령어로 변경할 수 있다.

```mcfunction
/allblocks change color red
```

플레이어는 모드 내부에서 사용할 표시 이름도 변경할 수 있다.

```mcfunction
/allblocks change name 원하는_이름
```

이 이름은 실제 마인크래프트 닉네임을 바꾸는 것이 아니라, All Blocks Challenge 내부 표시용 이름이다.
랭킹, 도감, 이벤트 메시지 등에 사용된다.

## 블록 수집 판정

블록 수집은 다음 기준으로 판정한다.

```text
플레이어 인벤토리에 블록 아이템이 새롭게 들어온 순간 도감 등록
```

중요 규칙은 다음과 같다.

* 플레이어 인벤토리에 들어온 블록만 인정한다.
* 상자, 엔더상자, 셜커상자, 호퍼 등에 들어있는 블록은 인정하지 않는다.
* 이미 다른 플레이어가 등록한 블록은 다시 등록되지 않는다.
* 사망 패널티로 해제된 블록은 누구나 다시 등록할 수 있다.
* 도감 점수는 현재 소유 중인 블록 개수를 기준으로 계산한다.

예시:

```text
A 플레이어가 참나무 원목을 최초 획득
→ 참나무 원목의 owner가 A로 저장됨
→ A의 점수 +1
→ 도감에서 참나무 원목 테두리가 A의 색상으로 표시됨
```

---

## 도감 시스템

도감은 전체 수집 대상 블록 목록을 보여준다.

각 블록은 다음 상태 중 하나를 가진다.

| 상태  | 설명                   |
| --- | -------------------- |
| 미획득 | 아무도 획득하지 않은 상태       |
| 획득됨 | 특정 플레이어가 현재 소유 중인 상태 |
| 해제됨 | 사망 패널티로 소유권이 풀린 상태   |

획득된 블록은 현재 소유자의 색상 테두리로 표시된다.

도감에서 블록에 마우스를 올리면 다음과 같이 표시한다.

```text
참나무 원목
준영
```

획득자가 없는 블록은 블록 이름만 표시한다.

```text
참나무 원목
```

사망 패널티로 해제된 블록은 일반 미획득 블록과 구분되도록 도감 배경보다 짙은 색의 테두리로 표시한다.

---

## 점수 및 랭킹

점수는 별도의 점수값을 저장하기보다, 각 블록의 현재 소유자를 기준으로 계산한다.

예시:

```text
stone → owner: playerA
dirt → owner: playerB
oak_log → owner: none
diamond_block → owner: playerA
```

이 경우 점수는 다음과 같다.

```text
playerA: 2점
playerB: 1점
```

즉, 도감에 등록된 블록 중 현재 owner가 해당 플레이어인 블록 개수가 곧 점수다.

사망 패널티로 블록 소유권이 해제되면 해당 플레이어의 점수도 함께 감소한다.

## HUD 구성

### Solo / Co-op

Solo와 Co-op에서는 전체 도감 진행률이 중요하므로 진행률 바를 표시한다.

```text
All Blocks Progress: 132 / 987
Day 17
```

### Team Race / Block Race

Team Race와 Block Race에서는 전체 진행률보다 경쟁 상황이 중요하므로, 플레이어 또는 팀별 점유율을 표시한다.

예시:

```text
A 38% | B 31% | C 21% | 미점유 10%
```

플레이어별 색상과 연동하여 줄다리기 형태의 점유율 바를 표시하는 것을 목표로 한다.

### 기타 HUD

추가로 표시할 수 있는 정보는 다음과 같다.

* 현재 Day
* 스톱워치
* 좌표
* 생물군계
* 인게임 시간
* 랭킹
* 도감 아이콘

모든 모드는 기본적으로 인게임 기준 100일 제한을 가진다.

* 플레이어별 점수

## 사망 패널티

플레이어가 사망하면, 해당 플레이어가 소유 중인 도감 블록 중 일부가 랜덤으로 해제된다.

사망 패널티는 다음과 같다.

| 사망 유형   | 손실률                   |
| ------- | --------------------- |
| PvP 사망  | 현재 소유 블록의 5~20% 랜덤 해제 |
| 비PvP 사망 | 현재 소유 블록의 0~10% 랜덤 해제 |

PvP 사망 여부는 게임의 사망 메시지 기준을 따른다.

예를 들어 플레이어에게 직접 죽은 것으로 표시되면 PvP 사망으로 처리하고, 용암, 낙사, 몬스터, 익사 등은 비PvP 사망으로 처리한다.

사망 패널티 예시:

```text
A 플레이어가 현재 50개 블록을 소유 중
PvP 사망 발생
손실률 10% 적용
→ 랜덤 블록 5개 해제
→ A의 점수 50점에서 45점으로 감소
```

해제된 블록은 누구나 다시 획득할 수 있다.

## 이벤트 시스템

블록 수집만 반복하면 지루해질 수 있으므로, 일정 조건마다 이벤트를 발생시킨다.

이벤트는 크게 두 종류다.

| 이벤트 종류       | 설명                       |
| ------------ | ------------------------ |
| Day 이벤트      | 10일마다 발생하는 생존 압박 이벤트     |
| Progress 이벤트 | 도감 진행률 10%마다 발생하는 방해 이벤트 |

---

## Day 이벤트

Day 이벤트는 10일마다 발생한다.

```text
Day 10
Day 20
Day 30
...
Day 100
```

Day 이벤트는 주로 적대 몬스터 소환, 웨이브, 생존 압박 요소를 중심으로 구성한다.

예시:

* 좀비 웨이브
* 스켈레톤 소환
* 강화 몬스터 소환
* 마녀 소환
* 약탈자 습격
* 보스급 몬스터 출현

---

## Progress 이벤트

Progress 이벤트는 전체 도감 진행률이 10% 단위에 도달할 때 발생한다.

```text
10%
20%
30%
...
100%
```

사망 패널티로 진행률이 떨어졌다가 다시 같은 구간에 도달해도 이벤트는 다시 발생할 수 있다.

예시:

```text
30% 도달
→ 30% 이벤트 발생

사망 패널티로 28% 하락
→ 다시 30% 도달
→ 30% 이벤트 다시 발생
```

이벤트는 해당 구간의 이벤트 풀에서 랜덤으로 선택된다.
같은 이벤트가 중복으로 나와도 허용한다.

Progress 이벤트는 주로 짧고 즉각적인 방해 요소를 중심으로 구성한다.

예시:

* 짧은 디버프
* 랜덤 텔레포트
* 5x5x5 범위 블록 삭제
* 5x5x5 범위 흑요석 대체
* 플레이어 주변에 자갈 낙하
* 일시적인 채굴 방해
* 시야 방해

## 경쟁 모드 이벤트

Team Race와 Block Race에서는 플레이어 간 경쟁 자체가 주요 변수이므로, Solo/Co-op보다 낮은 단계의 이벤트를 사용한다.

기본 방향은 다음과 같다.

```text
Solo / Co-op: 1~10단계 이벤트 사용
Team Race / Block Race: 1~5단계 이벤트 사용
```

즉, 경쟁 모드에서는 너무 강한 이벤트보다 짧고 가벼운 방해 이벤트를 중심으로 사용한다.

## Block Race

Block Race는 개인전 경쟁 모드다.

최종 우승자는 챌린지 종료 시점에 가장 많은 블록을 도감에 등록한 플레이어다.

Block Race의 핵심 특수 규칙은 드래곤 알을 이용한 조기 종료 시스템이다.

---

## Block Race: 드래곤 알 시스템

Block Race에서는 드래곤 알이 게임을 조기 종료시키는 핵심 오브젝트가 된다.

중요한 점은 다음과 같다.

```text
드래곤 알을 가진 사람 = 우승자 아님
드래곤 알을 가진 사람 = 조기 종료 타이머를 작동시키는 사람
최종 우승자 = 종료 시점에 도감 점수가 가장 높은 사람
```

---

## 드래곤 알 획득

Block Race 진행 중 플레이어가 드래곤 알 블록을 클릭하면 획득을 시도한다.

조건:

```text
플레이어 인벤토리에 빈 칸이 있어야 한다.
```

성공 시:

```text
드래곤 알 블록 제거
드래곤 알 아이템이 플레이어 인벤토리의 빈 칸에 직접 추가됨
해당 슬롯에 고정됨
Egg Carrier 상태 부여
10분 타이머 시작
```

실패 시:

```text
인벤토리가 꽉 차 있으면 아무 일도 일어나지 않음
드래곤 알 블록은 그대로 유지
타이머도 시작되지 않음
```

---

## 드래곤 알 고정

드래곤 알을 획득한 플레이어는 Egg Carrier가 된다.

Egg Carrier의 드래곤 알은 다음 행동이 불가능하다.

* 버리기
* 슬롯 이동
* 핫바 이동
* 인벤토리 내부 이동
* 상자에 넣기
* 셜커상자에 넣기
* 엔더상자에 넣기
* 기타 컨테이너에 보관

드래곤 알은 획득된 슬롯에 그대로 고정된다.

---

## 드래곤 알 10분 타이머

Egg Carrier가 드래곤 알을 10분 동안 보유하면 챌린지가 조기 종료된다.

```text
Egg Carrier가 10분 동안 생존하며 알을 보유
→ 챌린지 조기 종료
→ 종료 시점 랭킹 1위가 우승
```

Egg Carrier가 반드시 우승하는 것은 아니다.

---

## Egg Carrier 사망

Egg Carrier가 사망하면 다음 처리를 한다.

```text
드래곤 알 아이템 제거
Egg Carrier 상태 해제
10분 타이머 리셋
드래곤 알 블록을 안전한 위치에 재설치
```

드래곤 알은 아이템 엔티티로 드롭하지 않는다.
따라서 디스폰, 용암 소실, 폭발 소실 등의 문제를 방지한다.

---

## Egg Carrier 로그아웃

Egg Carrier가 로그아웃하면 다음 처리를 한다.

```text
드래곤 알 아이템 제거
Egg Carrier 상태 해제

```

## 드래곤 알 안전 위치 처리

드래곤 알은 절대 사라지면 안 되는 핵심 오브젝트다.

따라서 다음 상황에서는 안전한 위치로 이동하거나 재설치한다.

* Egg Carrier 사망
* Egg Carrier 로그아웃
* 드래곤 알이 용암, 불, 공허, 폭발 위험 위치에 있음
* 드래곤 알을 설치해야 하는 위치가 부적절함

안전 위치는 주변 블록을 검사하여 결정한다.

우선순위:

```text
1. 플레이어가 마지막으로 밟았던 안전한 위치
2. 현재 위치 주변의 안전한 지면
3. 스폰 지점 근처의 안전한 위치
```

---

## Egg Carrier 추적 흔적

Egg Carrier는 도망치는 동안 다른 플레이어에게 추적 단서를 남긴다.

방식:

```text
Egg Carrier가 밟고 지나간 블록 위에 노란빛 흔적 표시
흔적은 약 5분 동안 유지
흔적은 실제 블록을 변경하지 않고 시각적 효과로만 표시
```

이 흔적은 발자국처럼 이어져 다른 플레이어가 Egg Carrier를 추적할 수 있게 한다.

구현 시에는 렉 방지를 위해 매 틱마다 흔적을 남기지 않고, 일정 간격으로만 남긴다.

예상 규칙:

```text
1~2초마다 흔적 생성
같은 위치 또는 너무 가까운 위치에는 중복 생성 방지
약 5분 후 자동 제거
```

---

## Egg Carrier 차원 이동 제한

Egg Carrier는 드래곤 알을 들고 다른 차원으로 도망갈 수 없다.

규칙:

```text
드래곤 알 획득 전: 차원 이동 가능
드래곤 알 획득 후:
  - End → Overworld 이동 가능
  - Overworld → Nether 이동 불가
  - Overworld → End 이동 불가
  - Nether 이동 불가
```

드래곤 알은 엔드에서 얻기 때문에, 엔드에서 오버월드로 돌아오는 것은 허용한다.

## 수집 대상 블록

수집 대상은 생존 모드에서 실제로 획득 가능한 블록을 기준으로 한다.

기본 원칙:

```text
BlockItem이 존재하는 블록만 대상
생존에서 획득 불가능한 블록 제외
중복 블록 제거
```

제외 대상 예시:

* 기반암
* 명령 블록류
* 구조물 블록
* 구조물 공허
* 방벽
* 빛 블록
* 엔드 포탈 프레임
* 몬스터 생성기
* 수상한 모래
* 수상한 자갈
* 플레이어 머리
* 기타 생존에서 정상 획득 불가능한 블록

정확한 제외 목록은 테스트를 통해 확정한다.

## 현재 확정된 핵심 규칙

```text
1. 모드 이름은 Solo, Co-op, Team Race, Block Race로 한다.
2. Block Race는 도감 점수 개인전이다.
3. Block Race에서 드래곤 알은 조기 종료 오브젝트다.
4. 드래곤 알을 10분 동안 보유하면 챌린지가 조기 종료된다.
5. 조기 종료 시 우승자는 종료 시점의 도감 점수 1위다.
6. 드래곤 알은 인벤토리에 직접 들어오며, 아이템 엔티티로 드롭되지 않는다.
7. 드래곤 알은 획득된 슬롯에 고정된다.
8. 드래곤 알은 버리기, 이동, 컨테이너 보관이 불가능하다.
9. Egg Carrier가 죽거나 로그아웃하면 드래곤 알은 안전한 위치에 블록으로 재설치된다.
10. Egg Carrier는 노란빛 발자국 흔적을 남긴다.
11. 사망 시 PvP는 5~20%, 비PvP는 0~10%의 도감 블록을 잃는다.
12. 해제된 블록은 누구나 다시 획득할 수 있다.
13. Progress 이벤트는 재달성 시 반복 발생할 수 있다.
14. 경쟁 모드는 낮은 단계 이벤트를 중심으로 사용한다.
15. 챌린지 상태는 서버 재시작 후에도 유지되어야 한다.
```
